# RPG 2D 游戏项目

这是一个基于 Unity 引擎开发的 2D 横版 RPG 游戏项目。项目包含了一个完整的动作角色扮演系统，涵盖了角色控制、战斗、技能、背包、存档以及简单的 AI 系统。

## 核心系统介绍

### 1. 角色系统 (Entity & Player & Enemy)
- **状态机架构**: 所有角色（玩家和敌人）都基于状态机（State Machine）进行开发，确保了动作切换的流畅性和逻辑的可扩展性。
- **Entity 基类**: 提供了基础的移动、碰撞检测（地面、墙壁）、翻转、击退（Knockback）以及减速（Slowdown）等通用逻辑。
- **玩家控制**: 支持复杂的移动逻辑（二段跳、墙上滑行、冲刺）、多种攻击模式（基础攻击、跳跃攻击、反击、投掷剑）以及终极技能。
- **敌人 AI**: 包含基础的巡逻、追踪、战斗和被眩晕状态。敌人能够检测玩家并根据距离采取不同的行为。

### 2. 战斗与数值系统 (Combat & Stats)
- **数值驱动**: 角色属性（力量、敏捷、智力、生命值、防御、闪避等）通过 `Entity_Stats` 管理，并支持通过 ScriptableObjects 进行初始化。
- **伤害计算**: 系统支持物理伤害和元素伤害（火、冰、雷）。伤害计算考虑了暴击、护甲减免、元素抗性和闪避。
- **状态效果**: 攻击可以附带元素状态效果（如燃烧、冻结、感电），通过 `Entity_StatusHandller` 进行管理。
- **血量与再生**: 包含自动回血逻辑，并支持通过 UI 显示实时血量。

### 3. 技能系统 (Skill System)
- 玩家拥有多种技能，如 `SkillShard`（投掷碎片）和 `TimeEcho`（时间回溯/幻影）。技能通过 `Player_SkillManager` 统一调度。

### 4. 背包与装备系统 (Inventory & Equipment)
- **物品管理**: 包含基础物品、消耗品和装备。支持堆叠、使用和丢弃。
- **装备系统**: 玩家可以装备不同部位的防具和武器，装备会实时影响玩家的各项属性。
- **快捷栏**: 支持快捷键使用药品或道具。

### 5. 存档与加载系统 (Save System)
- **全自动存档**: 场景切换或退出游戏时自动保存。
- **数据涵盖**: 保存玩家位置、当前场景、金币、背包物品、已解锁的检查点等。
- **ISaveable 接口**: 任何需要保存数据的脚本只需实现 `ISaveable` 接口即可轻松集成到存档系统中。

### 6. 环境与交互 (Environment & Interaction)
- **视差滚动 (Parallax)**: 增强场景的深度感。
- **可交互物体**: 包含检查点（Checkpoints）、传送门（Portals）、路标（Waypoints）和宝箱。

## 逻辑问题与 Bug 修复报告

在代码审计过程中，我们发现并**修复**了以下问题：

### 已修复的关键问题：

1. **存档系统更新延迟 (`SaveManager.cs`)**: 
   - **问题**: `allSaveables` 列表仅在 `Start` 中初始化，切换场景后新生成的物体无法被自动识别。
   - **修复**: 增加了 `RefreshSaveables` 方法，并在每次保存和加载前强制刷新可存档物体列表。

2. **存档文件名拼写错误 (`SaveManager.cs`)**: 
   - **修复**: 将默认文件名从 `unityalexdev,json` 更正为 `unityalexdev.json`。

3. **血量 UI 更新潜在异常 (`Entity_Health.cs`)**: 
   - **问题**: `UpdateHealthBar` 中的空引用判断逻辑错误，会导致在 `healthBar` 为空时抛出异常。
   - **修复**: 修正了判断逻辑，确保在 `healthBar` 为空或未激活时安全退出。

4. **死亡判定逻辑 (`Entity_Health.cs`)**: 
   - **修复**: 将 `ReduceHealth` 中的死亡判定条件从 `< 0` 修改为 `<= 0`，并增加了 `!isDead` 的二次校验。

5. **战斗目标检测 Bug (`Entity_Combat.cs`)**: 
   - **问题**: `PerformAttackOnTarget` 函数忽略了传入的伤害倍率参数。
   - **修复**: 修正了函数调用，确保使用正确的 `damageScale`。

6. **敌人减速恢复逻辑 (`Enemy.cs`)**: 
   - **问题**: `StopSlowDown` 强行将动画速度设为 1，可能会覆盖敌人原本的动画缩放。
   - **修复**: 在 `Awake` 中记录初始动画速度，并在恢复时回退到该初始值。

7. **拼写错误修复**: 
   - 修正了 `Entity_Stats.cs` 中的 `StatType` 拼写错误。
   - 修正了 `Inventory_Player.cs` 中的 `itemToUse` 变量名拼写错误。

### Boss 特殊修复（Reaper）:

8. **Boss 施法状态管理 (`Enemy_Reaper.cs` & `Enemy_ReaperSpellCastState.cs`)**: 
   - **问题**: Boss 在施法时受到攻击会导致状态混乱，卡在施法动画。
   - **修复**: 添加了协程管理机制，确保施法过程可追踪，并在状态退出时正确清理。

9. **Boss 传送 Untargetable 状态 (`Enemy_ReaperTeleportState.cs`)**: 
   - **修复**: 实现了传送时的不可选中状态，避免在传送过程中被攻击打断。

10. **Boss 施法打断机制 (`Enemy.cs` & `Enemy_Reaper.cs`)**: 
   - **问题**: Boss 在施法时可以被玩家的攻击行为打断。
   - **修复**: 添加了 `CanBeInterrupted()` 虚拟方法，在施法状态下返回 false，确保施法不可被打断但仍可受到伤害。

## 项目架构特点

### 状态机模式
项目大量使用了状态机模式来管理角色行为，这种设计带来了以下优势：
- **清晰的逻辑分离**: 每个状态都有明确的进入、更新和退出逻辑
- **易于扩展**: 新状态可以通过继承轻松添加
- **避免状态冲突**: 状态切换由状态机统一管理，避免了复杂的条件判断

### 数据驱动设计
- 使用 ScriptableObjects 存储游戏数据，便于平衡调整和内容扩展
- 属性系统采用修饰器模式，支持运行时动态修改
- 存档系统基于接口设计，易于扩展新的可保存数据类型

### 模块化架构
- 各系统之间通过接口和事件进行解耦
- 支持功能的热插拔和独立测试
- 清晰的职责分离，便于团队协作开发

## 潜在的改进方向

基于代码质量检查，以下是一些建议的改进方向：

1. **性能优化**: 在大型场景中，敌人检测和物理计算可能需要优化
2. **代码一致性**: 部分命名规范和代码风格可以进一步统一
3. **错误处理**: 某些边界条件和异常情况的处理可以更加完善
4. **文档完善**: 核心系统的API文档和使用示例可以进一步丰富

## 技术栈

- **引擎**: Unity 2022.3 或更新版本
- **语言**: C# 
- **架构模式**: 状态机、观察者模式、修饰器模式
- **数据存储**: JSON 序列化
- **物理系统**: Unity 2D Physics
- **输入系统**: Unity Input System

这个项目展现了良好的软件工程实践，包括模块化设计、清晰的架构分层和完善的错误处理机制。通过状态机模式管理角色行为，使得代码结构清晰且易于维护。数据驱动的设计理念也让游戏平衡调整变得更加灵活。
